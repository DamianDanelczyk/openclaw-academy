# Cost Optimization

Let's talk about money. AI APIs aren't free, and if you're running OpenClaw for business, costs can sneak up on you faster than you'd expect. The good news? With some smart choices, you can cut your monthly bill by 60-80% without sacrificing quality where it matters.

This guide gives you real numbers, practical strategies, and a framework for deciding whether OpenClaw is worth the investment for your specific use case.

## Model Routing: The Single Biggest Cost Saver

The most impactful thing you can do is stop using one model for everything. Instead, **route each task to the cheapest model that can handle it well**.

Here's the key insight: roughly 70% of the tasks your agents handle are simple. Answering FAQs, classifying messages, extracting data from structured text, sending template responses. You don't need Opus for any of that. Haiku or GPT-4o Mini will do the job just as well at a fraction of the cost.

### Setting Up Model Routing

```yaml
# config/routing.yaml
model_routing:
  default: claude-haiku

  rules:
    - condition: "task_type == 'classification'"
      model: claude-haiku

    - condition: "task_type == 'simple_qa'"
      model: claude-haiku

    - condition: "task_type == 'content_generation' AND word_count < 200"
      model: claude-sonnet

    - condition: "task_type == 'content_generation' AND word_count >= 200"
      model: claude-sonnet

    - condition: "task_type == 'complex_reasoning'"
      model: claude-opus

    - condition: "task_type == 'code_review'"
      model: claude-sonnet

    - condition: "confidence_score < 0.7"
      model: claude-sonnet  # Escalate uncertain responses
```

The idea is simple: start cheap, escalate when needed. If Haiku isn't confident in its answer (confidence score below 0.7), bump it up to Sonnet. Reserve Opus for tasks that genuinely require deep reasoning.

### Automatic Complexity Detection

OpenClaw can analyze incoming requests and estimate complexity before choosing a model:

```yaml
# config/routing.yaml
complexity_detection:
  enabled: true
  method: keyword_and_length

  thresholds:
    simple:
      max_tokens: 100
      keywords: ["what is", "how do I", "where can I", "yes or no"]
      route_to: claude-haiku

    moderate:
      max_tokens: 500
      keywords: ["compare", "analyze", "explain why", "draft"]
      route_to: claude-sonnet

    complex:
      min_tokens: 500
      keywords: ["evaluate", "strategic", "multi-step", "trade-offs"]
      route_to: claude-opus
```

## Monthly Cost Estimates by Usage Level

Here's what real-world OpenClaw usage typically costs, assuming smart model routing is in place:

### Solo User / Personal Assistant

- **Volume:** ~1,000 tasks/month
- **Mix:** 80% Haiku, 15% Sonnet, 5% Opus
- **Estimated cost:** $5-15/month
- **Typical use:** Email summaries, scheduling, personal research

### Small Business

- **Volume:** ~10,000 tasks/month
- **Mix:** 70% Haiku, 25% Sonnet, 5% Opus
- **Estimated cost:** $30-80/month
- **Typical use:** Customer support bot, internal Q&A, content drafting

### Growing Business

- **Volume:** ~50,000 tasks/month
- **Mix:** 75% Haiku, 20% Sonnet, 5% Opus
- **Estimated cost:** $120-350/month
- **Typical use:** Multiple support channels, automated workflows, lead qualification

### Enterprise / Agency

- **Volume:** ~500,000 tasks/month
- **Mix:** 80% Haiku, 17% Sonnet, 3% Opus
- **Estimated cost:** $800-2,500/month
- **Typical use:** High-volume automation, multi-client deployments, complex pipelines

These numbers assume you've set up model routing. Without it — if you're running everything through Sonnet or Opus — multiply the cost by 5-10x.

## Break-Even Analysis

The real question isn't "how much does it cost?" It's "does it save me more than it costs?"

### Customer Support Example

Let's say you run a small e-commerce business. You currently handle 500 support tickets per month. A human support agent costs about $18/hour and handles roughly 8 tickets per hour.

**Without OpenClaw:**
- 500 tickets / 8 per hour = 62.5 hours
- 62.5 hours x $18/hour = **$1,125/month**

**With OpenClaw (handling 70% of tickets automatically):**
- 350 tickets handled by OpenClaw: ~$15/month in API costs
- 150 tickets still need a human: 18.75 hours x $18 = $337.50
- **Total: ~$353/month**

**Monthly savings: $772** (69% reduction)
**Break-even point: Immediately** — there's no upfront cost beyond setup time.

### Content Creation Example

You hire freelance writers for blog posts. Average cost: $200 per 1,500-word article. You publish 8 articles per month.

**Without OpenClaw:** 8 x $200 = **$1,600/month**

**With OpenClaw (drafting first versions that a human editor polishes):**
- 8 Sonnet-generated drafts: ~$2 in API costs
- Human editor (2 hours per article at $35/hr): $560
- **Total: ~$562/month**

**Monthly savings: $1,038.** You still get human oversight and quality control, but the bulk of the writing labor is handled by AI.

## Advanced Cost Strategies

Once you've got the basics in place, here are some additional ways to trim costs:

### 1. Response Caching

If your agents answer the same questions repeatedly, cache the responses.

```yaml
# config/caching.yaml
response_cache:
  enabled: true
  ttl: 24h  # Cache responses for 24 hours
  similarity_threshold: 0.92  # Match questions that are 92% similar
  max_entries: 10000
```

A well-configured cache can reduce API calls by 20-40% for support-style workloads where the same questions come up over and over.

### 2. Prompt Compression

Long system prompts eat tokens on every single request. A 2,000-token system prompt across 10,000 requests costs the same as 20 million input tokens. That's $5 with Haiku but $60 with Sonnet.

Keep system prompts lean. Move detailed knowledge into memory retrieval (RAG) so it's only loaded when needed, rather than stuffed into every prompt.

### 3. Batch Processing

If tasks aren't time-sensitive, batch them. Some APIs offer discounted pricing for batch requests, and even without that, batching lets you optimize token usage by grouping similar items together.

```yaml
# config/batch.yaml
batch_processing:
  enabled: true
  max_batch_size: 20
  max_wait_time: 5m
  applicable_tasks:
    - email_classification
    - daily_summaries
    - report_generation
```

### 4. Set Budgets and Alerts

Don't wait until the end of the month to discover you've overspent.

```yaml
# config/budget.yaml
budget:
  monthly_limit: 200.00
  currency: USD

  alerts:
    - threshold: 50%
      notify: email
    - threshold: 80%
      notify: email_and_slack
    - threshold: 95%
      action: switch_to_cheapest_models
    - threshold: 100%
      action: pause_non_critical_agents
```

The 95% threshold is especially useful — instead of cutting off service entirely, it automatically switches all agents to the cheapest available model. Quality drops slightly, but your business keeps running.

Most OpenClaw users who set up proper model routing spend between $30 and $300 per month. That's less than most SaaS subscriptions and almost always cheaper than the human labor it replaces.

The users who get hit with surprise bills are the ones who skip this guide, run everything through Opus, and never set up a budget alert. Don't be that person. Spend an hour on routing configuration now and save yourself a headache every month.
