# Gedächtnis

Gedächtnis ist das, was einen nützlichen Assistenten von einem vergesslichen Chatbot unterscheidet. OpenClaw verwendet einen **mehrschichtigen Gedächtnis-Stack**, der dem menschlichen Gedächtnis nachempfunden ist: Manche Dinge sind präsent, manche sind im Hinterkopf, und manche brauchen einen Auslöser, um erinnert zu werden. Diese Schichten zu verstehen hilft dir, das Beste aus der Fähigkeit deines Assistenten herauszuholen, sich zu erinnern — und zu vergessen.

## Der Gedächtnis-Stack

OpenClaws Gedächtnis ist in vier Schichten organisiert, jede mit einem anderen Zweck und einer anderen Lebensdauer:

```
┌─────────────────────────────┐
│      Sitzungsgedächtnis     │  ← Was du gerade gesagt hast (Minuten)
├─────────────────────────────┤
│      Tagesgedächtnis        │  ← Highlights des Tages (Stunden)
├─────────────────────────────┤
│      Langzeitgedächtnis     │  ← Wichtige Fakten (Wochen/Monate)
├─────────────────────────────┤
│      Vektorgedächtnis       │  ← Semantische Suche (für immer)
└─────────────────────────────┘
```

Jede Schicht speist die nächste. Was im Moment wichtig ist, lebt im Sitzungsgedächtnis. Was heute wichtig ist, wird ins Tagesgedächtnis befördert. Was langfristig wichtig ist, wird persistiert. Und alles ist über das Vektorgedächtnis durchsuchbar.

### Schicht 1: Sitzungsgedächtnis

Das ist das Arbeitsgedächtnis deines Assistenten — das aktuelle Gespräch. Jede Nachricht, jede Antwort, jedes Skill-Ergebnis lebt während einer aktiven Sitzung hier.

```yaml
session_memory:
  capacity: 128000 tokens
  strategy: "sliding_window"
  summarize_threshold: 50 messages
  priority_items:
    - user_instructions
    - active_tasks
    - recent_context
  eviction_policy: "oldest_first_with_summary"
```

Das Sitzungsgedächtnis ist schnell und detailliert, aber temporär. Wenn der Gesprächskontext zu groß wird, werden ältere Nachrichten zusammengefasst und komprimiert. Deine expliziten Anweisungen und aktiven Aufgaben werden nie entfernt — sie sind angeheftet.

**Was es erinnert:** Alles im aktuellen Gespräch.
**Wie lange:** Bis die Sitzung endet (normalerweise täglicher Reset).
**Beispiel:** "Du hast mich vor 10 Minuten gebeten, eine E-Mail zu entwerfen. Hier ist die aktualisierte Version."

### Schicht 2: Tagesgedächtnis

Am Ende jeder Sitzung destilliert OpenClaw die wichtigen Teile in das Tagesgedächtnis. Stell dir vor, dein Assistent schreibt in ein Tagebuch, bevor er schlafen geht.

```yaml
daily_memory:
  auto_generate: true
  max_entries_per_day: 20
  retention: "30 days"
  format:
    type: "structured"
    fields:
      - summary
      - decisions_made
      - tasks_completed
      - tasks_pending
      - preferences_learned
      - notable_events
```

So könnte ein Tagesgedächtnis-Eintrag aussehen:

```json
{
  "date": "2026-04-21",
  "summary": "Arbeitsreicher Tag, fokussiert auf Q2-Einführungsplanung.",
  "decisions_made": [
    "Entschieden, das Design-Review auf Donnerstag zu verschieben",
    "Option B für das Preismodell gewählt"
  ],
  "tasks_completed": [
    "E-Mail an Dr. Chen über den Zeitplan entworfen",
    "Konkurrenzpreise recherchiert"
  ],
  "tasks_pending": [
    "Bei Sarah wegen Mockups nachfragen",
    "Restaurant für Jamies Geburtstag buchen"
  ],
  "preferences_learned": [
    "Benutzer bevorzugt Tabellen gegenüber Aufzählungspunkten für Vergleiche"
  ]
}
```

**Was es erinnert:** Schlüsselereignisse, Entscheidungen und Aufgaben von jedem Tag.
**Wie lange:** 30 Tage standardmäßig (konfigurierbar).
**Beispiel:** "Gestern hast du dich für Option B beim Pricing entschieden."

### Schicht 3: Langzeitgedächtnis

Manche Dinge sind wichtiger als ein einzelner Tag. Das Langzeitgedächtnis speichert Fakten, Präferenzen und bedeutende Ereignisse, die dein Assistent auf unbestimmte Zeit kennen sollte.

```yaml
long_term_memory:
  storage: "local_db"
  categories:
    facts:
      description: "Konkrete Informationen über die Welt des Benutzers"
      examples: ["Sarah ist im Januar ins Team eingetreten", "Q2-Frist ist der 30. April"]
      retention: "until_invalidated"

    preferences:
      description: "Wie der Benutzer Dinge erledigt haben möchte"
      examples: ["Bevorzugt Morgentreffen", "Mag detaillierte Code-Kommentare"]
      retention: "permanent"

    relationships:
      description: "Personen und wie sie mit dem Benutzer verbunden sind"
      examples: ["Mike berichtet an Damian", "Dr. Chen ist VP of Product"]
      retention: "until_updated"

    events:
      description: "Bedeutende Ereignisse, die passiert sind"
      examples: ["Im März 2026 befördert worden", "Projekt Falcon gestartet"]
      retention: "permanent"

  management:
    deduplication: true
    conflict_resolution: "latest_wins"
    max_entries: 10000
```

Langzeiterinnerungen sind strukturiert und getaggt. Wenn dein Agent am Anfang einer Sitzung Kontext lädt, ruft er die relevantesten Langzeiterinnerungen basierend auf aktuellen Themen ab.

**Was es erinnert:** Wichtige Fakten, Präferenzen, Beziehungen und Meilensteine.
**Wie lange:** Unbegrenzt, bis explizit aktualisiert oder entfernt.
**Beispiel:** "Dein Partner Jamies Geburtstag ist am 15. März. Soll ich etwas planen?"

### Schicht 4: Vektorgedächtnis

Das ist das tiefe Archiv. Jede Interaktion, die du je hattest, wird als Vektor (eine mathematische Bedeutungsdarstellung) eingebettet und in einer durchsuchbaren Datenbank gespeichert. Dein Agent lädt all das nicht in den Kontext — das wäre unmöglich groß. Stattdessen *sucht* er in dieser Schicht, wenn er etwas Bestimmtes finden muss.

```yaml
vector_memory:
  engine: "local_embeddings"
  model: "text-embedding-3-small"
  dimensions: 1536
  index:
    type: "hnsw"
    metric: "cosine"
  search:
    default_top_k: 5
    score_threshold: 0.75
    rerank: true
  storage:
    location: "~/.openclaw/vectors/"
    max_size: "5GB"
    compression: true
```

Wenn du fragst "Was habe ich letzten Monat über Preismodelle gesagt?" — erinnert dein Agent sich daran nicht aus dem Sitzungs- oder Tagesgedächtnis. Er führt eine semantische Suche über dein Vektorgedächtnis durch, findet die relevanten Gespräche und bringt sie an die Oberfläche.

**Was es erinnert:** Den semantischen Inhalt aller vergangenen Interaktionen.
**Wie lange:** Für immer (bis du es löschst).
**Beispiel:** "In einem Gespräch am 12. März hast du drei Preismodelle besprochen. Hier ist, was du über jedes gesagt hast..."

## Gedächtnis in Aktion

Hier ist, wie alle vier Schichten in einer realen Interaktion zusammenarbeiten:

```
Du: Was soll ich wegen der Lieferantensituation tun?

OpenClaws interner Prozess:
1. Sitzungsgedächtnis: "Der Benutzer hat vor 20 Minuten ein Lieferantenproblem erwähnt"
2. Tagesgedächtnis: "Gestern hat der Benutzer entschieden, dem Lieferanten noch eine Woche Zeit zu geben"
3. Langzeitgedächtnis: "Dieser Lieferant (Acme Corp) hat seit Februar Lieferpobleme"
4. Vektorsuche: Findet 3 vergangene Gespräche über Acme Corp Lieferantenprobleme

Antwort: Basierend auf unseren Gesprächen hatte Acme Corp seit Februar wiederkehrende
Lieferprobleme. Gestern hast du ihnen noch eine Woche gegeben. Diese Frist wäre
nächsten Montag. Soll ich die Folge-E-Mail entwerfen, oder sollten wir anfangen,
alternative Lieferanten zu prüfen?
```

Keine einzelne Gedächtnisschicht hätte diese Antwort produzieren können. Es war Kontext aus allen vier nötig.

## Gedächtnis-Verwaltungsbefehle

Du hast volle Kontrolle über das Gedächtnis deines Agenten:

```bash
# Gedächtnis-Statistiken sehen
openclaw memory stats

# Ausgabe:
# Session: 23 Nachrichten (18% Kapazität)
# Daily: 147 Einträge (letzte 30 Tage)
# Long-term: 892 Fakten gespeichert
# Vector: 2,1 GB (12.340 Einbettungen)

# Erinnerungen suchen
openclaw memory search "Lieferantenpreise"

# Etwas Bestimmtes vergessen
openclaw memory forget --query "Gehaltsverhandlungsdetails"

# Alle Erinnerungen exportieren
openclaw memory export --format json

# Vektorgedächtnis komprimieren (entfernt redundante Einbettungen)
openclaw memory compact

# Aufbewahrungsrichtlinie setzen
openclaw memory retention --daily 60d --vector unlimited
```

## Gedächtnisstrategien

### Der Minimalist

Gedächtnis schlank halten. Gut für datenschutzbewusste Benutzer.

```yaml
strategy: "minimalist"
session_memory: true
daily_memory:
  enabled: true
  retention: "7 days"
long_term_memory:
  enabled: true
  categories: ["preferences"]  # Nur Präferenzen merken
vector_memory:
  enabled: false
```

### Der Elefant

Alles merken. Gut für Power-User, die tiefen Abruf möchten.

```yaml
strategy: "elephant"
session_memory: true
daily_memory:
  enabled: true
  retention: "365 days"
long_term_memory:
  enabled: true
  categories: ["all"]
  max_entries: 50000
vector_memory:
  enabled: true
  max_size: "20GB"
  retention: "unlimited"
```

### Der Profi

Balance zwischen Nützlichkeit und Datenschutz. Merkt Arbeitssachen, vergisst persönliche Plaudereien.

```yaml
strategy: "professional"
session_memory: true
daily_memory:
  enabled: true
  retention: "90 days"
  filter:
    include: ["tasks", "decisions", "meetings"]
    exclude: ["casual", "personal"]
long_term_memory:
  enabled: true
  categories: ["facts", "relationships", "preferences"]
vector_memory:
  enabled: true
  max_size: "5GB"
  filter:
    include_topics: ["work", "projects", "professional"]
```

## Wann man aufräumen sollte

Gedächtnis ist nicht immer besser mit mehr Daten. Alte, veraltete Erinnerungen können deinen Agenten tatsächlich verwirren. Hier sind Anzeichen, dass es Zeit zum Aufräumen ist:

- Dein Agent referenziert Informationen, die nicht mehr aktuell sind.
- Suchergebnisse liefern zu viele irrelevante Treffer.
- Dein Vektorspeicher wächst über 5 GB ohne klaren Nutzen.
- Dein Agent scheint "verwirrt" über Fakten, die sich geändert haben.

Führe `openclaw memory audit` aus, um einen Bericht über veraltete und möglicherweise überholte Erinnerungen zu erhalten, dann überprüfe und bereinige nach Bedarf.

Gedächtnis ist das Rückgrat eines personalisierten Assistenten. Richtig eingerichtet fühlt sich dein Agent an, als würde er dich wirklich kennen. Unkontrolliert wird er zu einem überfüllten Dachboden. Die gute Nachricht? Ein paar Minuten Konfiguration geht einen langen Weg.
