# Canal de Telegram

Telegram es uno de los canales más fáciles y con más funciones para configurar con OpenClaw. La plataforma tiene una API de bots bien documentada, límites de velocidad generosos y soporte para formato enriquecido de mensajes. Si buscas una forma rápida de lanzar tu asistente al mundo real, Telegram es una excelente primera opción.

Esta guía te lleva paso a paso desde la creación de un bot de Telegram hasta conectarlo a OpenClaw y configurar el canal para uso en producción.

## Paso 1: Crear Tu Bot con BotFather

Cada bot de Telegram comienza con una conversación con [@BotFather](https://t.me/BotFather), el bot oficial de gestión de bots de Telegram. Abre Telegram, busca BotFather e inicia un chat.

Envía el siguiente comando:

```
/newbot
```

BotFather te hará dos preguntas:

1. **Un nombre de visualización** para tu bot (por ejemplo, "OpenClaw Assistant"). Puede contener espacios y es lo que los usuarios ven en el encabezado del chat.
2. **Un nombre de usuario** para tu bot (por ejemplo, `openclaw_assistant_bot`). Debe terminar en `bot` y no puede contener espacios. Es el `@usuario` que la gente usa para encontrar y mencionar tu bot.

Una vez que respondas ambas, BotFather te dará un **token de API** — una cadena larga que se parece a algo así:

```
7123456789:AAHfB3xJ9kWvQ_mNpL8r2s5tUvWxYz1a2b3
```

Copia este token y guárdalo en un lugar seguro. Cualquiera con este token puede controlar tu bot.

### Personalizando Tu Bot

Mientras chateas con BotFather, tómate un momento para pulir el perfil de tu bot:

```
/setdescription   — The text users see before they start a chat
/setabouttext     — The short bio visible in the bot's profile
/setuserpic       — Upload a profile photo
/setcommands      — Define the command menu (more on this below)
```

Para el menú de comandos, puedes configurar accesos rápidos útiles:

```
/setcommands

help - Show available commands
reset - Start a fresh conversation
status - Check assistant status
settings - View your preferences
```

Estos comandos aparecen como un menú cuando los usuarios escriben `/` en el chat.

## Paso 2: Configurar OpenClaw

Agrega el canal de Telegram a tu configuración de OpenClaw:

```typescript title="openclaw.config.ts"
import { defineConfig } from '@openclaw/core';

export default defineConfig({
  channels: {
    telegram: {
      enabled: true,
      token: process.env.TELEGRAM_BOT_TOKEN,

      // Choose one: 'webhook' (recommended for production)
      // or 'polling' (easier for development)
      mode: 'webhook',

      webhook: {
        url: 'https://your-server.com/hooks/telegram',
        secret: process.env.TELEGRAM_WEBHOOK_SECRET, // Optional but recommended
        port: 3100, // Local port OpenClaw listens on for webhook delivery
      },

      // Bot behavior
      commands: {
        help: 'I can answer your questions, summarize documents, and more. Just type naturally!',
        reset: async (ctx) => {
          await ctx.clearHistory();
          return 'Conversation cleared. Let us start fresh!';
        },
      },
    },
  },
});
```

Y tus variables de entorno:

```bash title=".env"
TELEGRAM_BOT_TOKEN=7123456789:AAHfB3xJ9kWvQ_mNpL8r2s5tUvWxYz1a2b3
TELEGRAM_WEBHOOK_SECRET=a-random-secret-string-you-generate
```

## Webhook vs. Sondeo

OpenClaw admite dos modos para recibir mensajes de Telegram. Así es como elegir:

### Sondeo (Desarrollo)

En el modo de sondeo, OpenClaw pregunta repetidamente a Telegram "¿hay nuevos mensajes?" cada pocos segundos. Es sencillo, no requiere una URL pública y funciona detrás de firewalls.

```typescript
telegram: {
  enabled: true,
  token: process.env.TELEGRAM_BOT_TOKEN,
  mode: 'polling',
  polling: {
    interval: 1500,  // Milliseconds between polls
    timeout: 30,     // Long-polling timeout in seconds
  },
},
```

Usa el sondeo cuando estés desarrollando localmente y no quieras lidiar con túneles o URLs públicas.

### Webhook (Producción)

En el modo webhook, Telegram envía actualizaciones a tu servidor en el momento en que llegan. Es más rápido, más eficiente y el enfoque recomendado para todo lo que supere las pruebas locales.

```typescript
telegram: {
  enabled: true,
  token: process.env.TELEGRAM_BOT_TOKEN,
  mode: 'webhook',
  webhook: {
    url: 'https://your-server.com/hooks/telegram',
    secret: process.env.TELEGRAM_WEBHOOK_SECRET,
  },
},
```

OpenClaw registra el webhook con Telegram automáticamente al inicio. Puedes verificar que se configuró correctamente:

```bash
curl https://api.telegram.org/bot<YOUR_TOKEN>/getWebhookInfo
```

Deberías ver tu URL en la respuesta junto con `"has_custom_certificate": false` y `"pending_update_count": 0`.

## El Framework grammY Bajo el Capó

El canal de Telegram de OpenClaw está construido sobre [grammY](https://grammy.dev), un framework moderno con TypeScript primero para bots de Telegram. Por lo general, no necesitas interactuar directamente con grammY — OpenClaw lo abstrae — pero saber que está ahí es útil cuando quieres hacer cosas avanzadas.

Por ejemplo, puedes acceder a la instancia del bot de grammY para middleware personalizado:

```typescript title="openclaw.config.ts"
telegram: {
  enabled: true,
  token: process.env.TELEGRAM_BOT_TOKEN,
  mode: 'webhook',
  advanced: {
    onBotCreated: (bot) => {
      // 'bot' is a grammY Bot instance
      // Add custom middleware before OpenClaw's handlers
      bot.use(async (ctx, next) => {
        console.log(`Message from ${ctx.from?.first_name}: ${ctx.message?.text}`);
        await next();
      });
    },
  },
},
```

Esto te da el poder completo del ecosistema de grammY — plugins para limitación de velocidad, gestión de sesiones, menús, teclados en línea y más — mientras dejas que OpenClaw maneje la interacción de IA central.

## Formato de Mensajes

Telegram admite formato de texto enriquecido, y OpenClaw lo aprovecha. Por defecto, las respuestas del asistente se envían usando el formato MarkdownV2 de Telegram:

- **Texto en negrita** para énfasis
- `Bloques de código` para contenido técnico
- Listas con viñetas y numeradas
- [Enlaces en línea](https://example.com) para referencias

Puedes controlar el comportamiento del formato:

```typescript
telegram: {
  formatting: {
    parseMode: 'MarkdownV2',  // 'MarkdownV2' | 'HTML' | 'None'
    codeBlockLanguage: true,   // Syntax-highlight code blocks
    linkPreviews: false,       // Disable link preview cards
  },
},
```

> **Aviso:** El MarkdownV2 de Telegram es notoriamente exigente con el escape de caracteres especiales (puntos, guiones, paréntesis, etc.). OpenClaw maneja este escape automáticamente, por lo que no necesitas preocuparte por ello en las respuestas de tu asistente.

## Configuración de Chats Grupales

Los bots de Telegram pueden participar en chats grupales, pero por lo general no quieres que respondan a cada mensaje individual. Así es como controlar el comportamiento del grupo:

```typescript
telegram: {
  groups: {
    enabled: true,
    respondTo: 'mention',    // Only respond when @mentioned
    allowedGroups: [         // Optional: restrict to specific groups
      -1001234567890,
      -1009876543210,
    ],
    adminOnly: false,        // If true, only group admins can interact
  },
},
```

Las opciones de `respondTo` son:

- `'mention'` — Solo responder cuando el bot es mencionado con `@` por nombre de usuario (recomendado).
- `'reply'` — Responder a las respuestas directas a los mensajes del bot.
- `'prefix'` — Responder a mensajes que comienzan con un prefijo configurable como `/ask` o `!claw`.
- `'all'` — Responder a todos los mensajes del grupo (usa con precaución en grupos activos).

### Modo de Privacidad

Por defecto, los bots de Telegram en grupos solo reciben mensajes que los mencionan o son comandos. Esto se llama **modo de privacidad** y está habilitado de forma predeterminada. Si necesitas que el bot vea todos los mensajes (para el modo de respuesta `'all'`), debes deshabilitar el modo de privacidad a través de BotFather:

```
/setprivacy
@your_bot_username
Disable
```

## Manejo de Medios

El canal de Telegram admite medios enriquecidos en ambas direcciones:

**Recepción de usuarios:**
- Fotos (comprimidas y sin comprimir)
- Videos y notas de video (videos redondos)
- Mensajes de voz (transcritos automáticamente a texto)
- Documentos (PDFs, hojas de cálculo, etc.)
- Stickers (convertidos a su equivalente emoji)
- Ubicaciones compartidas (analizadas como coordenadas)

**Envío a usuarios:**
- Imágenes con subtítulos
- Documentos como adjuntos de archivo
- Mensajes de voz (si tu asistente genera audio)

El manejo de medios está habilitado de forma predeterminada. Los archivos entrantes se descargan temporalmente, los procesa el motor central y se limpian después.

## Teclados en Línea y Mensajes Interactivos

Puedes configurar tu asistente para enviar botones interactivos a través de la función de teclado en línea de Telegram:

```typescript
telegram: {
  interactivity: {
    enabled: true,
    confirmActions: true,  // Ask for confirmation before destructive actions
    feedbackButtons: true, // Add thumbs up/down to every response
  },
},
```

Cuando `feedbackButtons` está habilitado, cada respuesta del asistente obtiene una pequeña fila de botones de reacción. Esta retroalimentación se registra y puede usarse para mejorar tu asistente con el tiempo.

## Límites de Velocidad y Cuotas

Los límites de velocidad de Telegram son generosos pero reales:

- **Chats privados:** 30 mensajes por segundo en todos los chats.
- **Chats grupales:** 20 mensajes por minuto por grupo.
- **Notificaciones masivas:** Máximo 30 usuarios por segundo para mensajes de difusión.

La cola integrada de OpenClaw respeta estos límites automáticamente. Si los alcanzas, los mensajes se almacenan en búfer y se envían tan pronto como la ventana se abre. Verás un registro como:

```
[telegram] Rate limit approaching, throttling outbound messages (24/30 used)
```

## Consejos por Entorno

**Desarrollo:** Usa el modo de sondeo. No se necesita túnel ni URL pública.

**Staging:** Usa un bot separado (crea otro con BotFather) para que el tráfico de staging no interfiera con la producción.

**Producción:** Usa el modo webhook con un certificado TLS válido. Establece el campo `secret` para que las solicitudes de Telegram sean verificadas. Considera colocar un proxy inverso (nginx, Caddy) delante del endpoint de webhook para mayor seguridad.

```typescript title="Production-ready config"
telegram: {
  enabled: true,
  token: process.env.TELEGRAM_BOT_TOKEN,
  mode: 'webhook',
  webhook: {
    url: 'https://api.yourcompany.com/hooks/telegram',
    secret: process.env.TELEGRAM_WEBHOOK_SECRET,
    maxConnections: 40,
  },
  rateLimit: {
    windowMs: 60000,
    maxPerUser: 20, // Prevent individual users from flooding the bot
  },
},
```

Telegram es un canal fantástico para OpenClaw — rápido de configurar, rico en funciones y confiable en producción. Una vez que esté funcionando, te preguntarás cómo probabas tu asistente sin él.
