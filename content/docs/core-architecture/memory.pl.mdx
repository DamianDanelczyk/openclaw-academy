# Pamięć

Pamięć to to, co odróżnia użytecznego asystenta od zapominalskiego chatbota. OpenClaw używa **warstwowego stosu pamięci**, który odzwierciedla działanie ludzkiej pamięci: jedne rzeczy są na pierwszym planie, inne w tle, a jeszcze inne wymagają bodźca, żeby je przywołać. Zrozumienie tych warstw pozwala w pełni wykorzystać zdolność asystenta do zapamiętywania — i zapominania.

## Stos Pamięci

Pamięć OpenClaw jest zorganizowana w cztery warstwy, z których każda ma inny cel i czas trwania:

```
┌─────────────────────────────┐
│      Session Memory         │  ← Co właśnie powiedziałeś (minuty)
├─────────────────────────────┤
│      Daily Memory           │  ← Najważniejsze z dnia (godziny)
├─────────────────────────────┤
│      Long-Term Memory       │  ← Ważne fakty (tygodnie/miesiące)
├─────────────────────────────┤
│      Vector Memory          │  ← Semantyczne wyszukiwanie (na zawsze)
└─────────────────────────────┘
```

Każda warstwa zasila kolejną. To, co ma znaczenie w danej chwili, trafia do pamięci sesji. To, co ma znaczenie dzisiaj, awansuje do pamięci dziennej. To, co ma długotrwałe znaczenie, jest utrwalane. A wszystko jest przeszukiwalne przez pamięć wektorową.

### Warstwa 1: Pamięć Sesji

To jest robocza pamięć asystenta — bieżąca rozmowa. Każda wiadomość, każda odpowiedź, każdy wynik umiejętności żyje tu podczas aktywnej sesji.

```yaml
session_memory:
  capacity: 128000 tokens
  strategy: "sliding_window"
  summarize_threshold: 50 messages
  priority_items:
    - user_instructions
    - active_tasks
    - recent_context
  eviction_policy: "oldest_first_with_summary"
```

Pamięć sesji jest szybka i szczegółowa, ale tymczasowa. Gdy kontekst rozmowy staje się zbyt duży, starsze wiadomości są podsumowywane i kompresowane. Twoje wyraźne instrukcje i aktywne zadania nigdy nie są usuwane — są przypiete.

**Co zapamiętuje:** Wszystko w bieżącej rozmowie.
**Jak długo:** Do zakończenia sesji (zazwyczaj reset dzienny).
**Przykład:** „Poprosiłeś mnie o napisanie e-maila 10 minut temu. Oto zaktualizowana wersja."

### Warstwa 2: Pamięć Dzienna

Na koniec każdej sesji OpenClaw destyluje ważne części do pamięci dziennej. Pomyśl o tym jak o asystencie piszącym w dzienniku przed snem.

```yaml
daily_memory:
  auto_generate: true
  max_entries_per_day: 20
  retention: "30 days"
  format:
    type: "structured"
    fields:
      - summary
      - decisions_made
      - tasks_completed
      - tasks_pending
      - preferences_learned
      - notable_events
```

Oto jak może wyglądać wpis w pamięci dziennej:

```json
{
  "date": "2026-04-21",
  "summary": "Busy work day focused on Q2 launch planning.",
  "decisions_made": [
    "Decided to push the design review to Thursday",
    "Chose Option B for the pricing model"
  ],
  "tasks_completed": [
    "Drafted email to Dr. Chen about timeline",
    "Researched competitor pricing"
  ],
  "tasks_pending": [
    "Follow up with Sarah about mockups",
    "Book restaurant for Jamie's birthday"
  ],
  "preferences_learned": [
    "User prefers tables over bullet lists for comparisons"
  ]
}
```

**Co zapamiętuje:** Kluczowe zdarzenia, decyzje i zadania z każdego dnia.
**Jak długo:** 30 dni domyślnie (konfigurowalne).
**Przykład:** „Wczoraj zdecydowałeś się na Opcję B dla cennika."

### Warstwa 3: Pamięć Długoterminowa

Niektóre rzeczy mają znaczenie dłużej niż jeden dzień. Pamięć długoterminowa przechowuje fakty, preferencje i znaczące zdarzenia, o których Twój asystent powinien wiedzieć bezterminowo.

```yaml
long_term_memory:
  storage: "local_db"
  categories:
    facts:
      description: "Concrete information about the user's world"
      examples: ["Sarah joined the team in January", "Q2 deadline is April 30"]
      retention: "until_invalidated"

    preferences:
      description: "How the user likes things done"
      examples: ["Prefers morning meetings", "Likes detailed code comments"]
      retention: "permanent"

    relationships:
      description: "People and how they connect to the user"
      examples: ["Mike reports to Damian", "Dr. Chen is VP of Product"]
      retention: "until_updated"

    events:
      description: "Significant things that happened"
      examples: ["Got promoted in March 2026", "Launched Project Falcon"]
      retention: "permanent"

  management:
    deduplication: true
    conflict_resolution: "latest_wins"
    max_entries: 10000
```

Wspomnienia długoterminowe są ustrukturyzowane i oznaczone tagami. Gdy Twój agent ładuje kontekst na początku sesji, pobiera najbardziej odpowiednie wspomnienia długoterminowe na podstawie ostatnich tematów.

**Co zapamiętuje:** Ważne fakty, preferencje, relacje i kamienie milowe.
**Jak długo:** Bezterminowo, dopóki nie zostanie wyraźnie zaktualizowane lub usunięte.
**Przykład:** „Urodziny Twojego partnera Jamiego przypadają 15 marca. Chcesz coś zaplanować?"

### Warstwa 4: Pamięć Wektorowa

To jest głębokie archiwum. Każda interakcja, którą kiedykolwiek miałeś, jest osadzona jako wektor (matematyczna reprezentacja znaczenia) i przechowywana w przeszukiwalnej bazie danych. Twój agent nie ładuje tego wszystkiego do kontekstu — byłoby to niemożliwie duże. Zamiast tego *przeszukuje* tę warstwę, gdy musi coś konkretnego znaleźć.

```yaml
vector_memory:
  engine: "local_embeddings"
  model: "text-embedding-3-small"
  dimensions: 1536
  index:
    type: "hnsw"
    metric: "cosine"
  search:
    default_top_k: 5
    score_threshold: 0.75
    rerank: true
  storage:
    location: "~/.openclaw/vectors/"
    max_size: "5GB"
    compression: true
```

Gdy zapytasz „Co mówiłem o modelach cenowych w zeszłym miesiącu?" — Twój agent nie pamięta tego z pamięci sesji ani dziennej. Przeprowadza semantyczne wyszukiwanie w swojej pamięci wektorowej, znajduje odpowiednie rozmowy i wyświetla je.

**Co zapamiętuje:** Semantyczną zawartość wszystkich przeszłych interakcji.
**Jak długo:** Na zawsze (dopóki tego nie usuniesz).
**Przykład:** „W rozmowie 12 marca omawiałeś trzy modele cenowe. Oto co mówiłeś o każdym z nich..."

## Pamięć w Działaniu

Oto jak wszystkie cztery warstwy współpracują w rzeczywistej interakcji:

```
Ty: Co powinienem zrobić w sprawie dostawcy?

Wewnętrzny proces OpenClaw:
1. Pamięć sesji: "Użytkownik wspomniał o problemie z dostawcą 20 minut temu"
2. Pamięć dzienna: "Wczoraj użytkownik dał dostawcy jeszcze tydzień"
3. Pamięć długoterminowa: "Ten dostawca (Acme Corp) ma problemy z dostawą od lutego"
4. Wyszukiwanie wektorowe: Znajduje 3 poprzednie rozmowy o problemach z dostawcą Acme Corp

Odpowiedź: Na podstawie naszych rozmów Acme Corp ma powtarzające się problemy
z dostawą od lutego. Wczoraj dałeś im jeszcze tydzień. Ten termin
przypada w przyszły poniedziałek. Czy chcesz, żebym napisał e-mail
z podsumowaniem, czy powinniśmy zacząć szukać alternatywnych dostawców?
```

Żadna pojedyncza warstwa pamięci nie mogłaby wygenerować tej odpowiedzi. Wymagała kontekstu ze wszystkich czterech.

## Polecenia Zarządzania Pamięcią

Masz pełną kontrolę nad pamięcią agenta:

```bash
# Wyświetl statystyki pamięci
openclaw memory stats

# Wyjście:
# Session: 23 messages (18% capacity)
# Daily: 147 entries (last 30 days)
# Long-term: 892 facts stored
# Vector: 2.1GB (12,340 embeddings)

# Wyszukaj wspomnienia
openclaw memory search "vendor pricing"

# Zapomnij coś konkretnego
openclaw memory forget --query "salary negotiation details"

# Eksportuj wszystkie wspomnienia
openclaw memory export --format json

# Kompaktuj pamięć wektorową (usuwa redundantne osadzenia)
openclaw memory compact

# Ustaw politykę retencji
openclaw memory retention --daily 60d --vector unlimited
```

## Strategie Pamięci

### Minimalistyczna

Utrzymuj pamięć w ryzach. Dobra dla użytkowników dbających o prywatność.

```yaml
strategy: "minimalist"
session_memory: true
daily_memory:
  enabled: true
  retention: "7 days"
long_term_memory:
  enabled: true
  categories: ["preferences"]  # Zapamiętuj tylko preferencje
vector_memory:
  enabled: false
```

### Słoń

Zapamiętuj wszystko. Dobra dla zaawansowanych użytkowników, którzy chcą głębokiego przywołania.

```yaml
strategy: "elephant"
session_memory: true
daily_memory:
  enabled: true
  retention: "365 days"
long_term_memory:
  enabled: true
  categories: ["all"]
  max_entries: 50000
vector_memory:
  enabled: true
  max_size: "20GB"
  retention: "unlimited"
```

### Zawodowa

Balans między użytecznością a prywatnością. Zapamiętuje sprawy zawodowe, zapomina luźne rozmowy.

```yaml
strategy: "professional"
session_memory: true
daily_memory:
  enabled: true
  retention: "90 days"
  filter:
    include: ["tasks", "decisions", "meetings"]
    exclude: ["casual", "personal"]
long_term_memory:
  enabled: true
  categories: ["facts", "relationships", "preferences"]
vector_memory:
  enabled: true
  max_size: "5GB"
  filter:
    include_topics: ["work", "projects", "professional"]
```

## Kiedy Przycinać

Pamięć nie zawsze jest lepsza z większą ilością danych. Stare, przestarzałe wspomnienia mogą w rzeczywistości mylić agenta. Oto oznaki, że czas na przycinanie:

- Twój agent odwołuje się do informacji, które nie są już dokładne.
- Wyniki wyszukiwania zwracają zbyt wiele nieistotnych dopasowań.
- Twoje repozytorium wektorowe rośnie powyżej 5 GB bez wyraźnych korzyści.
- Twój agent wydaje się „zdezorientowany" co do faktów, które się zmieniły.

Uruchom `openclaw memory audit`, aby otrzymać raport o przestarzałych i potencjalnie nieaktualnych wspomnieniach, a następnie przejrzyj i wyczyść według potrzeb.

Pamięć to kręgosłup spersonalizowanego asystenta. Skonfiguruj ją poprawnie, a Twój agent będzie sprawiał wrażenie, że naprawdę Cię zna. Pozostaw ją niezarządzaną, a stanie się zagraconym stryszkiem. Dobra wiadomość? Kilka minut konfiguracji wystarczy.
