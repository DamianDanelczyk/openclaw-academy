# Umiejętności i ClawHub

Fabrycznie OpenClaw potrafi rozmawiać. Dzięki **umiejętnościom** potrafi *działać*. Umiejętności to modułowe możliwości, które pozwalają Twojemu agentowi wchodzić w interakcje ze światem zewnętrznym — sprawdzać kalendarz, wysyłać e-maile, przeszukiwać internet, sterować urządzeniami inteligentnego domu, uruchamiać kod i setki innych rzeczy. **ClawHub** to marketplace, gdzie je znajdziesz, zainstalujujesz i udostępnisz.

## Czym Jest Umiejętność?

Umiejętność to samodzielny pakiet, który daje Twojemu agentowi nową zdolność. Definiuje:

- **Co potrafi robić** — działania, które obsługuje
- **Czego potrzebuje** — wszelkie klucze API, uprawnienia lub konfiguracje
- **Kiedy jej używać** — warunki wyzwalania i dopasowywanie intencji

Gdy mówisz „Co mam dzisiaj w kalendarzu?", Twój agent nie zna magicznie Twojego harmonogramu. Umiejętność `calendar` łączy się z Google Calendar (lub Outlook, albo Apple Calendar), pobiera Twoje wydarzenia i zwraca je w formacie, który Twój agent może Ci przedstawić.

## Instalowanie Umiejętności z ClawHub

Instalacja umiejętności to jedna linijka:

```bash
# Zainstaluj jedną umiejętność
openclaw skill install calendar

# Zainstaluj wiele umiejętności naraz
openclaw skill install calendar email web_search notes

# Zainstaluj konkretną wersję
openclaw skill install github@2.4.1

# Przeglądaj dostępne umiejętności
openclaw skill search "project management"

# Sprawdź, co jest zainstalowane
openclaw skill list
```

Po instalacji większość umiejętności wymaga konfiguracji — zazwyczaj klucza API lub połączenia z kontem:

```bash
# Skonfiguruj umiejętność interaktywnie
openclaw skill setup calendar
```

Uruchamia to przewodnika, który przeprowadzi Cię przez połączenie konta, ustawienie uprawnień i przetestowanie połączenia.

### Plik Konfiguracyjny Umiejętności

Pod spodem konfiguracje umiejętności żyją w pliku YAML:

```yaml
# skills/calendar.yaml
skill: calendar
version: "3.2.0"
source: "clawhub/official"

auth:
  provider: "google"
  credentials: "${GOOGLE_OAUTH_TOKEN}"
  scopes:
    - "calendar.readonly"
    - "calendar.events"

settings:
  default_calendar: "Work"
  time_format: "24h"
  timezone: "Europe/Warsaw"
  reminder_default: 15  # minuty przed wydarzeniem

permissions:
  can_create_events: true
  can_delete_events: false  # Bezpieczeństwo: wymagaj potwierdzenia
  can_invite_others: true
  can_access_other_calendars: false

triggers:
  - intent: "check_schedule"
    examples: ["what's on my calendar", "any meetings today", "am I free at 3"]
  - intent: "create_event"
    examples: ["schedule a meeting", "add to my calendar", "book time for"]
```

Zwróć uwagę na ustawienie `can_delete_events: false`. Możesz kontrolować dokładnie to, co umiejętność może robić, nawet jeśli umiejętność *obsługuje* więcej. Tak właśnie zapobiegasz przypadkowemu usunięciu przez agenta spotkania o 15:00.

## Budowanie Własnych Umiejętności

ClawHub ma setki umiejętności, ale czasem potrzebujesz czegoś specyficznego dla swojego przepływu pracy. Budowanie własnej umiejętności jest proste.

### Struktura Umiejętności

```
my-custom-skill/
├── manifest.yaml      # Metadane i konfiguracja
├── handler.py         # Główna logika
├── triggers.yaml      # Kiedy umiejętność się aktywuje
├── test/
│   └── test_handler.py
└── README.md
```

### Manifest

```yaml
# manifest.yaml
name: "jira-standup"
version: "1.0.0"
author: "damian"
description: "Generates daily standup summaries from Jira"
runtime: "python3.11"

dependencies:
  - jira-client>=3.0
  - markdown2>=2.4

auth:
  required:
    - name: "JIRA_API_TOKEN"
      description: "Your Jira API token"
    - name: "JIRA_URL"
      description: "Your Jira instance URL"

permissions:
  network: true        # Musi wywoływać Jira API
  filesystem: false    # Nie potrzebuje dostępu do plików
  execute: false       # Brak wykonywania powłoki
```

### Handler

```python
# handler.py
from openclaw_sdk import Skill, SkillResponse
from jira import JIRA

class JiraStandupSkill(Skill):
    def setup(self):
        self.jira = JIRA(
            server=self.config["JIRA_URL"],
            token_auth=self.config["JIRA_API_TOKEN"]
        )

    def handle(self, intent: str, params: dict) -> SkillResponse:
        if intent == "generate_standup":
            return self.generate_standup(params.get("user", "currentUser()"))

    def generate_standup(self, user: str) -> SkillResponse:
        # Pobierz zgłoszenia zaktualizowane wczoraj
        yesterday_issues = self.jira.search_issues(
            f'assignee = {user} AND updated >= -1d ORDER BY updated DESC'
        )

        # Pobierz zgłoszenia bieżącego sprintu
        in_progress = self.jira.search_issues(
            f'assignee = {user} AND status = "In Progress"'
        )

        standup = {
            "completed_yesterday": [
                {"key": i.key, "summary": i.fields.summary}
                for i in yesterday_issues
                if i.fields.status.name == "Done"
            ],
            "working_on_today": [
                {"key": i.key, "summary": i.fields.summary}
                for i in in_progress
            ],
            "blockers": [
                {"key": i.key, "summary": i.fields.summary}
                for i in in_progress
                if hasattr(i.fields, 'flagged') and i.fields.flagged
            ]
        }

        return SkillResponse(
            data=standup,
            display="standup_template",
            summary=f"Found {len(standup['completed_yesterday'])} completed, "
                    f"{len(standup['working_on_today'])} in progress"
        )
```

## Zagadnienia Bezpieczeństwa

Umiejętności mogą być potężne, co oznacza, że potrzebują zabezpieczeń.

### Model Uprawnień

Każda umiejętność deklaruje, czego potrzebuje, a Ty zatwierdzasz lub odrzucasz:

```yaml
# Czego umiejętność żąda vs. co przyznasz
skill: smart_home
requested_permissions:
  network: true        # Musi rozmawiać z centrum inteligentnego domu
  filesystem: false
  execute: false
  location: true       # Chce lokalizacji do geofencingu

granted_permissions:
  network: true        # Zatwierdzone
  location: false      # Odrzucone — nie chcę geofencingu
```

### Piaskownica

Umiejętności działają w izolowanych piaskownicach. Nieprawidłowo działająca umiejętność nie może:

- Uzyskiwać dostępu do danych innych umiejętności
- Bezpośrednio odczytywać Twojego USER.md lub SOUL.md
- Wykonywać wywołań sieciowych poza zadeklarowanymi punktami końcowymi
- Wykonywać poleceń systemowych (chyba że wyraźnie dozwolone)
- Utrwalać danych poza przydzielonym jej magazynem

```yaml
# Polityka bezpieczeństwa dla umiejętności
security:
  sandbox:
    enabled: true
    network_allowlist:
      calendar: ["*.googleapis.com"]
      email: ["*.gmail.com", "*.outlook.com"]
      smart_home: ["192.168.1.0/24"]  # Tylko sieć lokalna
    resource_limits:
      max_memory: "256MB"
      max_cpu_time: "10s"
      max_storage: "100MB"
  audit:
    log_all_actions: true
    alert_on_permission_escalation: true
  review:
    require_review_for: ["filesystem", "execute"]
    auto_approve: ["network"]  # Jeśli na liście dozwolonych
```

### Weryfikacja na ClawHub

Umiejętności na ClawHub przechodzą przez proces recenzji:

- **Oficjalne umiejętności** (od zespołu OpenClaw) są w pełni audytowane.
- **Zweryfikowane umiejętności** (od znanych deweloperów) są poddane recenzji kodu.
- **Umiejętności społeczności** są poddane wzajemnej recenzji i oznaczane, gdy pojawią się problemy.
- **Prywatne umiejętności** pomijają recenzję (tylko Ty ich używasz).

Zawsze sprawdzaj uprawnienia umiejętności przed instalacją. Jeśli umiejętność pogodowa prosi o dostęp do systemu plików, to czerwona flaga.

## Wskazówki dla Maksymalnego Wykorzystania Umiejętności

1. **Zacznij od czterech podstawowych:** calendar, email, web_search i notes pokrywają 80% potrzeb większości ludzi.
2. **Konfiguruj uprawnienia ściśle.** Jeśli umiejętność może usuwać rzeczy, rozważ wyłączenie tego, dopóki jej nie zaufasz.
3. **Sprawdzaj aktualizacje co miesiąc.** Umiejętności się poprawiają, błędy są naprawiane, a łatki bezpieczeństwa są wydawane.
4. **Buduj zanim przeglądasz.** Jeśli Twój przepływ pracy jest unikalny, własna umiejętność dopasowana do perfekcji bije ogólną, która jakoś tam działa.
5. **Czytaj uprawnienia.** Pięć sekund czytania może zapobiec dostępowi umiejętności do rzeczy, do których nie powinna mieć dostępu.
