# Memoria

La memoria es lo que separa un asistente útil de un chatbot olvidadizo. OpenClaw usa una **pila de memoria en capas** que refleja cómo funciona la memoria humana: algunas cosas están en la mente en el momento, otras están en el fondo de tu cabeza y otras requieren un desencadenante para recordarlas. Entender estas capas te permite sacar el máximo provecho de la capacidad de recordar — y olvidar — de tu asistente.

## La Pila de Memoria

La memoria de OpenClaw está organizada en cuatro capas, cada una con un propósito y vida útil diferente:

```
┌─────────────────────────────┐
│      Memoria de Sesión      │  ← Lo que acabas de decir (minutos)
├─────────────────────────────┤
│      Memoria Diaria         │  ← Los aspectos destacados de hoy (horas)
├─────────────────────────────┤
│      Memoria a Largo Plazo  │  ← Hechos importantes (semanas/meses)
├─────────────────────────────┤
│      Memoria Vectorial      │  ← Búsqueda semántica (para siempre)
└─────────────────────────────┘
```

Cada capa alimenta a la siguiente. Lo que importa en el momento vive en la memoria de sesión. Lo que importa hoy se promueve a la memoria diaria. Lo que importa a largo plazo se persiste. Y todo es buscable a través de la memoria vectorial.

### Capa 1: Memoria de Sesión

Esta es la memoria de trabajo de tu asistente — la conversación actual. Cada mensaje, cada respuesta, cada resultado de habilidad vive aquí durante una sesión activa.

```yaml
session_memory:
  capacity: 128000 tokens
  strategy: "sliding_window"
  summarize_threshold: 50 messages
  priority_items:
    - user_instructions
    - active_tasks
    - recent_context
  eviction_policy: "oldest_first_with_summary"
```

La memoria de sesión es rápida y detallada pero temporal. Cuando el contexto de la conversación se vuelve demasiado grande, los mensajes más antiguos se resumen y comprimen. Tus instrucciones explícitas y tareas activas nunca se eliminan — están ancladas.

**Qué recuerda:** Todo en la conversación actual.
**Por cuánto tiempo:** Hasta que la sesión termina (generalmente reinicio diario).
**Ejemplo:** "Me pediste que redactara un correo electrónico hace 10 minutos. Aquí está la versión actualizada."

### Capa 2: Memoria Diaria

Al final de cada sesión, OpenClaw destila las partes importantes en la memoria diaria. Piensa en ello como tu asistente escribiendo en un diario antes de dormir.

```yaml
daily_memory:
  auto_generate: true
  max_entries_per_day: 20
  retention: "30 days"
  format:
    type: "structured"
    fields:
      - summary
      - decisions_made
      - tasks_completed
      - tasks_pending
      - preferences_learned
      - notable_events
```

Así es como podría verse una entrada de memoria diaria:

```json
{
  "date": "2026-04-21",
  "summary": "Día de trabajo ocupado enfocado en la planificación del lanzamiento del T2.",
  "decisions_made": [
    "Decidió posponer la revisión de diseño al jueves",
    "Eligió la Opción B para el modelo de precios"
  ],
  "tasks_completed": [
    "Redactó correo electrónico al Dr. Chen sobre el cronograma",
    "Investigó precios de la competencia"
  ],
  "tasks_pending": [
    "Hacer seguimiento con Sarah sobre las maquetas",
    "Reservar restaurante para el cumpleaños de Jamie"
  ],
  "preferences_learned": [
    "El usuario prefiere tablas sobre listas con viñetas para comparaciones"
  ]
}
```

**Qué recuerda:** Eventos clave, decisiones y tareas de cada día.
**Por cuánto tiempo:** 30 días por defecto (configurable).
**Ejemplo:** "Ayer decidiste optar por la Opción B para precios."

### Capa 3: Memoria a Largo Plazo

Algunas cosas importan más allá de un solo día. La memoria a largo plazo almacena hechos, preferencias y eventos significativos que tu asistente debería saber indefinidamente.

```yaml
long_term_memory:
  storage: "local_db"
  categories:
    facts:
      description: "Información concreta sobre el mundo del usuario"
      examples: ["Sarah se unió al equipo en enero", "El plazo del T2 es el 30 de abril"]
      retention: "until_invalidated"

    preferences:
      description: "Cómo al usuario le gusta que se hagan las cosas"
      examples: ["Prefiere reuniones por la mañana", "Le gustan los comentarios de código detallados"]
      retention: "permanent"

    relationships:
      description: "Personas y cómo se conectan con el usuario"
      examples: ["Mike reporta a Damian", "El Dr. Chen es VP de Producto"]
      retention: "until_updated"

    events:
      description: "Cosas significativas que sucedieron"
      examples: ["Fue ascendido en marzo de 2026", "Lanzó el Proyecto Falcon"]
      retention: "permanent"

  management:
    deduplication: true
    conflict_resolution: "latest_wins"
    max_entries: 10000
```

Las memorias a largo plazo están estructuradas y etiquetadas. Cuando tu agente carga contexto al inicio de una sesión, extrae las memorias a largo plazo más relevantes basándose en los temas recientes.

**Qué recuerda:** Hechos importantes, preferencias, relaciones e hitos.
**Por cuánto tiempo:** Indefinidamente, hasta que se actualice o elimine explícitamente.
**Ejemplo:** "El cumpleaños de tu pareja Jamie es el 15 de marzo. ¿Quieres que planifique algo?"

### Capa 4: Memoria Vectorial

Este es el archivo profundo. Cada interacción que hayas tenido alguna vez está incrustada como un vector (una representación matemática del significado) y almacenada en una base de datos buscable. Tu agente no carga todo esto en el contexto — eso sería imposiblemente grande. En cambio, *busca* en esta capa cuando necesita encontrar algo específico.

```yaml
vector_memory:
  engine: "local_embeddings"
  model: "text-embedding-3-small"
  dimensions: 1536
  index:
    type: "hnsw"
    metric: "cosine"
  search:
    default_top_k: 5
    score_threshold: 0.75
    rerank: true
  storage:
    location: "~/.openclaw/vectors/"
    max_size: "5GB"
    compression: true
```

Cuando preguntas "¿Qué dije sobre los modelos de precios el mes pasado?" — tu agente no recuerda esto desde la memoria de sesión o diaria. Realiza una búsqueda semántica en tu memoria vectorial, encuentra las conversaciones relevantes y las muestra.

**Qué recuerda:** El contenido semántico de todas las interacciones pasadas.
**Por cuánto tiempo:** Para siempre (hasta que lo elimines).
**Ejemplo:** "En una conversación del 12 de marzo, discutiste tres modelos de precios. Aquí está lo que dijiste sobre cada uno..."

## La Memoria en Acción

Así es como las cuatro capas trabajan juntas en una interacción real:

```
Tú: ¿Qué debería hacer con la situación del proveedor?

Proceso interno de OpenClaw:
1. Memoria de sesión: "El usuario mencionó un problema con el proveedor hace 20 minutos"
2. Memoria diaria: "Ayer, el usuario decidió darle al proveedor una semana más"
3. Memoria a largo plazo: "Este proveedor (Acme Corp) ha tenido problemas de entrega desde febrero"
4. Búsqueda vectorial: Encuentra 3 conversaciones pasadas sobre problemas con el proveedor Acme Corp

Respuesta: Basándome en nuestras conversaciones, Acme Corp ha tenido
problemas de entrega recurrentes desde febrero. Ayer les diste una semana más.
Ese plazo sería el próximo lunes. ¿Quieres que redacte el correo de seguimiento,
o deberíamos empezar a buscar proveedores alternativos?
```

Ninguna capa de memoria por sí sola podría haber producido esa respuesta. Requirió contexto de las cuatro.

## Comandos de Gestión de Memoria

Tienes control total sobre la memoria de tu agente:

```bash
# Ver estadísticas de memoria
openclaw memory stats

# Salida:
# Session: 23 mensajes (18% de capacidad)
# Daily: 147 entradas (últimos 30 días)
# Long-term: 892 hechos almacenados
# Vector: 2.1GB (12,340 incrustaciones)

# Buscar memorias
openclaw memory search "precios del proveedor"

# Olvidar algo específico
openclaw memory forget --query "detalles de negociación salarial"

# Exportar todas las memorias
openclaw memory export --format json

# Compactar memoria vectorial (elimina incrustaciones redundantes)
openclaw memory compact

# Establecer política de retención
openclaw memory retention --daily 60d --vector unlimited
```

## Estrategias de Memoria

### El Minimalista

Mantén la memoria reducida. Bueno para usuarios enfocados en la privacidad.

```yaml
strategy: "minimalist"
session_memory: true
daily_memory:
  enabled: true
  retention: "7 days"
long_term_memory:
  enabled: true
  categories: ["preferences"]  # Solo recordar preferencias
vector_memory:
  enabled: false
```

### El Elefante

Recuerda todo. Bueno para usuarios avanzados que quieren un recuerdo profundo.

```yaml
strategy: "elephant"
session_memory: true
daily_memory:
  enabled: true
  retention: "365 days"
long_term_memory:
  enabled: true
  categories: ["all"]
  max_entries: 50000
vector_memory:
  enabled: true
  max_size: "20GB"
  retention: "unlimited"
```

### El Profesional

Equilibrio entre utilidad y privacidad. Recuerda cosas de trabajo, olvida las charlas personales.

```yaml
strategy: "professional"
session_memory: true
daily_memory:
  enabled: true
  retention: "90 days"
  filter:
    include: ["tasks", "decisions", "meetings"]
    exclude: ["casual", "personal"]
long_term_memory:
  enabled: true
  categories: ["facts", "relationships", "preferences"]
vector_memory:
  enabled: true
  max_size: "5GB"
  filter:
    include_topics: ["work", "projects", "professional"]
```

## Cuándo Podar

La memoria no siempre es mejor con más datos. Las memorias antiguas y desactualizadas pueden en realidad confundir a tu agente. Aquí hay señales de que es hora de podar:

- Tu agente hace referencia a información que ya no es precisa.
- Los resultados de búsqueda devuelven demasiadas coincidencias irrelevantes.
- Tu almacén vectorial está creciendo más allá de 5GB sin un beneficio claro.
- Tu agente parece "confundido" sobre hechos que han cambiado.

Ejecuta `openclaw memory audit` para obtener un informe sobre memorias obsoletas y potencialmente desactualizadas, luego revisa y limpia según sea necesario.

La memoria es la columna vertebral de un asistente personalizado. Hazlo bien y tu agente sentirá que realmente te conoce. Déjala sin gestionar y se convierte en un desván abarrotado. La buena noticia es que unos pocos minutos de configuración hacen mucho.
