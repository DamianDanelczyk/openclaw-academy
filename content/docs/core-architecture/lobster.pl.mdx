# Lobster

Niektóre zadania nie są jednorazowe. Są sekwencjami — „gdy to się stanie, zrób tamto, potem sprawdź to, a następnie powiadom mnie." Robienie tego ręcznie za każdym razem jest żmudne. Dlatego OpenClaw zawiera **Lobster** — wizualny kreator przepływów pracy, który pozwala automatyzować wieloetapowe procesy bez pisania kodu.

## Czym Jest Lobster?

Lobster to kreator przepływów pracy metodą przeciągnij i upuść wbudowany w OpenClaw. Jeśli kiedykolwiek używałeś narzędzi takich jak Zapier, Make lub Apple Shortcuts, koncepcja jest podobna — ale z kluczową różnicą: Twój agent AI jest pełnoprawnym uczestnikiem przepływu pracy. Przepływy pracy Lobster to nie tylko automatyzacje „jeśli to, to tamto". Mogą obejmować rozumowanie agenta, decyzje i przetwarzanie języka naturalnego na każdym etapie.

Pomyśl o tym tak: tradycyjne narzędzia do automatyzacji łączą aplikacje. Lobster łączy aplikacje *i* Twojego agenta AI.

## Jak Działa Lobster

Każdy przepływ pracy Lobster to seria **kroków** połączonych **przepływami**. Kroki to działania — wyślij e-mail, sprawdź kalendarz, poproś agenta o podsumowanie czegoś. Przepływy to linie między nimi — mogą być proste (zawsze idź do następnego kroku) lub warunkowe (idź w lewo jeśli X, w prawo jeśli Y).

```
┌─────────┐     ┌──────────┐     ┌──────────┐
│ Trigger │ ──→ │  Step 1   │ ──→ │  Step 2   │
└─────────┘     └──────────┘     └──────────┘
                                       │
                               ┌───────┴───────┐
                               │  Condition?    │
                               └───────┬───────┘
                              Yes │         │ No
                                  ▼         ▼
                            ┌─────────┐ ┌─────────┐
                            │ Step 3a │ │ Step 3b │
                            └─────────┘ └─────────┘
```

Budujesz je wizualnie w edytorze Lobster — przeciągasz kroki na canvas, łączysz je liniami, konfigurujesz ustawienia każdego kroku.

## Konfiguracja Przepływu Pracy

Chociaż przepływy pracy możesz budować wizualnie, są one przechowywane jako YAML pod spodem. Oznacza to, że możesz je też pisać ręcznie, kontrolować wersje lub udostępniać innym.

```yaml
# lobster/workflows/example.yaml
workflow:
  name: "Morning Briefing"
  description: "Automated morning summary delivered daily"
  enabled: true

  trigger:
    type: "schedule"
    cron: "0 8 * * 1-5"  # 8 AM, Monday through Friday
    timezone: "Europe/Warsaw"

  steps:
    - id: "get_calendar"
      skill: "calendar"
      action: "get_today_events"
      output: "today_events"

    - id: "get_weather"
      skill: "weather"
      action: "get_forecast"
      params:
        location: "Warsaw"
      output: "weather"

    - id: "get_emails"
      skill: "email"
      action: "get_unread"
      params:
        limit: 10
        priority: "high"
      output: "urgent_emails"

    - id: "summarize"
      type: "agent"
      prompt: |
        Create a morning briefing from this data:
        Calendar: {{today_events}}
        Weather: {{weather}}
        Urgent emails: {{urgent_emails}}

        Keep it under 200 words. Start with the most important item.
      output: "briefing"

    - id: "deliver"
      type: "notify"
      channel: "voice"
      message: "{{briefing}}"
      fallback: "push_notification"
```

To jest kompletny przepływ pracy. Każdego dnia roboczego o 8 rano Lobster sprawdza Twój kalendarz, pogodę i e-mail, prosi agenta o stworzenie podsumowania i odczytuje je Ci. Jeśli głos nie jest dostępny (może Twój telefon jest wyciszony), przełącza się na powiadomienie push.

## 5 Przykładowych Przepływów Pracy

### 1. Segregacja E-maili

Automatycznie sortuj przychodzące e-maile według pilności i twórz szkice odpowiedzi na rutynowe.

```yaml
workflow:
  name: "Email Triage"
  description: "Sort and pre-respond to incoming emails"

  trigger:
    type: "event"
    source: "email"
    event: "new_email"
    filter:
      exclude_senders: ["noreply@*", "newsletter@*"]

  steps:
    - id: "classify"
      type: "agent"
      prompt: |
        Classify this email:
        From: {{trigger.sender}}
        Subject: {{trigger.subject}}
        Body: {{trigger.body}}

        Categories: urgent, needs_response, informational, spam
        Also rate importance 1-5.
      output: "classification"

    - id: "route"
      type: "condition"
      rules:
        - if: "classification.category == 'urgent'"
          goto: "alert_user"
        - if: "classification.category == 'needs_response'"
          goto: "draft_response"
        - if: "classification.category == 'informational'"
          goto: "file_away"
        - if: "classification.category == 'spam'"
          goto: "archive"

    - id: "alert_user"
      type: "notify"
      channel: "push"
      priority: "high"
      message: "Urgent email from {{trigger.sender}}: {{trigger.subject}}"

    - id: "draft_response"
      type: "agent"
      prompt: |
        Draft a brief, professional response to this email.
        Match my usual tone and style.
        From: {{trigger.sender}}
        Subject: {{trigger.subject}}
        Body: {{trigger.body}}
      output: "draft"

    - id: "save_draft"
      skill: "email"
      action: "save_draft"
      params:
        to: "{{trigger.sender}}"
        subject: "Re: {{trigger.subject}}"
        body: "{{draft}}"

    - id: "file_away"
      skill: "email"
      action: "move_to_folder"
      params:
        folder: "Read Later"

    - id: "archive"
      skill: "email"
      action: "archive"
```

Ten przepływ pracy przechwytuje każdy przychodzący e-mail (z wyjątkiem newsletterów i adresów noreply). Agent klasyfikuje go, a Lobster go kieruje: pilne e-maile dostają natychmiastowe powiadomienie push, e-maile wymagające odpowiedzi mają zapisany wstępnie napisany szkic, informacyjne trafiają do folderu Do Przeczytania Później, a spam jest archiwizowany. Budzisz się ze schludną skrzynką odbiorczą.

### 2. Przygotowanie do Spotkania

Automatycznie przygotowuj dokumenty briefingowe przed każdym spotkaniem.

```yaml
workflow:
  name: "Meeting Prep"
  description: "Prepare context before each meeting"

  trigger:
    type: "event"
    source: "calendar"
    event: "upcoming_event"
    lead_time: 15          # Trigger 15 minutes before

  steps:
    - id: "get_attendees"
      skill: "calendar"
      action: "get_event_details"
      params:
        event_id: "{{trigger.event_id}}"
      output: "event_details"

    - id: "find_context"
      type: "agent"
      prompt: |
        I have a meeting in 15 minutes:
        Title: {{event_details.title}}
        Attendees: {{event_details.attendees}}
        Agenda: {{event_details.description}}

        Search my memories for relevant context about these people
        and this topic. Prepare a brief (under 150 words) with:
        - Key points from last meeting with these people
        - Any pending action items
        - Suggested talking points
      memory_search: true
      output: "briefing"

    - id: "check_docs"
      skill: "file_manager"
      action: "search"
      params:
        query: "{{event_details.title}}"
        recency: "30 days"
      output: "related_docs"

    - id: "deliver"
      type: "notify"
      channel: "push"
      message: |
        Meeting in 15 min: {{event_details.title}}

        {{briefing}}

        Related docs: {{related_docs.names}}
```

### 3. Generator Raportów Tygodniowych

Kompiluj tygodniowe podsumowanie każdego piątku po południu.

```yaml
workflow:
  name: "Weekly Report"
  description: "Generate and send weekly progress report"

  trigger:
    type: "schedule"
    cron: "0 16 * * 5"   # 4 PM every Friday
    timezone: "Europe/Warsaw"

  steps:
    - id: "get_completed_tasks"
      skill: "jira"
      action: "get_completed"
      params:
        period: "this_week"
        assignee: "me"
      output: "completed"

    - id: "get_commits"
      skill: "github"
      action: "get_activity"
      params:
        period: "this_week"
        user: "me"
      output: "code_activity"

    - id: "get_meetings"
      skill: "calendar"
      action: "get_events"
      params:
        period: "this_week"
      output: "meetings"

    - id: "generate_report"
      type: "agent"
      prompt: |
        Write a weekly report based on:
        Completed tasks: {{completed}}
        Code activity: {{code_activity}}
        Meetings attended: {{meetings}}

        Format:
        ## Accomplishments
        ## In Progress
        ## Blockers
        ## Next Week
      output: "report"

    - id: "render_canvas"
      type: "canvas"
      template: "weekly_report"
      data:
        tasks: "{{completed}}"
        report_text: "{{report}}"
      output: "visual_report"

    - id: "send_report"
      skill: "email"
      action: "send"
      params:
        to: "dr.chen@techcorp.com"
        subject: "Weekly Report — {{date.format('MMM D')}}"
        body: "{{report}}"
        attachments: ["{{visual_report.export_pdf}}"]
```

### 4. Rutyna Inteligentnego Domu

Zautomatyzuj swoje wieczorne wyciszenie.

```yaml
workflow:
  name: "Evening Wind-Down"
  description: "Gradually transition the home for sleep"

  trigger:
    type: "schedule"
    cron: "0 21 * * *"   # 9 PM daily
    alternative_trigger:
      type: "voice"
      phrase: "start wind-down"

  steps:
    - id: "dim_lights"
      skill: "smart_home"
      action: "set_lights"
      params:
        rooms: ["living_room", "bedroom"]
        brightness: 30
        color_temp: "warm"
        transition: "10 minutes"

    - id: "adjust_thermostat"
      skill: "smart_home"
      action: "set_temperature"
      params:
        target: 19
        mode: "night"

    - id: "check_tomorrow"
      skill: "calendar"
      action: "get_tomorrow_events"
      output: "tomorrow"

    - id: "prepare_summary"
      type: "agent"
      prompt: |
        Give me a very brief tomorrow preview:
        Events: {{tomorrow}}
        Keep it to 2-3 sentences. Mention if I need to wake up
        earlier than usual for an early meeting.
      output: "preview"

    - id: "set_alarm"
      type: "condition"
      rules:
        - if: "tomorrow.earliest_event.time < '09:00'"
          goto: "early_alarm"
        - else:
          goto: "normal_alarm"

    - id: "early_alarm"
      skill: "reminders"
      action: "set_alarm"
      params:
        time: "{{tomorrow.earliest_event.time - 90min}}"
        label: "Early meeting day"
      goto: "goodnight"

    - id: "normal_alarm"
      skill: "reminders"
      action: "set_alarm"
      params:
        time: "07:30"
        label: "Good morning"

    - id: "goodnight"
      type: "notify"
      channel: "voice"
      message: "{{preview}} Your alarm is set. Goodnight."
      voice_settings:
        voice: "flint"
        volume: 0.3
```

### 5. Potok Badań Treści

Zbadaj temat, skompiluj źródła i utwórz ustrukturyzowany brief.

```yaml
workflow:
  name: "Research Pipeline"
  description: "Deep research on a given topic"

  trigger:
    type: "command"
    phrase: "/research {topic}"

  input:
    topic:
      type: "string"
      required: true

  steps:
    - id: "web_search"
      skill: "web_search"
      action: "search"
      params:
        query: "{{input.topic}}"
        num_results: 20
        recency: "6 months"
      output: "search_results"

    - id: "filter_sources"
      type: "agent"
      prompt: |
        Review these search results for "{{input.topic}}":
        {{search_results}}

        Select the 5 most credible and relevant sources.
        Explain why each was selected.
      output: "top_sources"

    - id: "deep_read"
      skill: "web_search"
      action: "read_pages"
      params:
        urls: "{{top_sources.urls}}"
      output: "full_content"

    - id: "synthesize"
      type: "agent"
      prompt: |
        Based on these sources about "{{input.topic}}":
        {{full_content}}

        Create a research brief with:
        ## Key Findings (3-5 bullet points)
        ## Detailed Analysis (500 words)
        ## Conflicting Viewpoints (if any)
        ## Sources (with links)
        ## Questions for Further Research
      output: "brief"

    - id: "save"
      skill: "notes"
      action: "create"
      params:
        title: "Research: {{input.topic}}"
        content: "{{brief}}"
        tags: ["research", "{{input.topic}}"]

    - id: "notify"
      type: "notify"
      channel: "push"
      message: "Research on '{{input.topic}}' is complete. Saved to your notes."
```

Wywołaj to za pomocą `/research quantum computing trends` i odejdź. Wróć do ustrukturyzowanego briefu w swoich notatkach.

## Zarządzanie Przepływami Pracy

```bash
# Lista wszystkich przepływów pracy
openclaw lobster list

# Uruchom przepływ pracy ręcznie
openclaw lobster run "Morning Briefing"

# Wstrzymaj przepływ pracy
openclaw lobster pause "Email Triage"

# Wyświetl historię uruchomień
openclaw lobster history "Weekly Report" --last 10

# Debuguj nieudane uruchomienie
openclaw lobster debug --run-id run_abc123

# Zaimportuj udostępniony przepływ pracy
openclaw lobster import workflow.yaml

# Eksportuj do udostępniania
openclaw lobster export "Meeting Prep" --output meeting-prep.yaml
```

## Obsługa Błędów

Przepływy pracy mogą się nie powieść — API może być niedostępne, umiejętność może przekroczyć limit czasu. Lobster obsługuje to elegancko:

```yaml
workflow:
  error_handling:
    default: "retry_then_skip"
    retries: 3
    retry_delay: 10s

    on_failure:
      notify: true
      channel: "push"
      message: "Workflow '{{workflow.name}}' failed at step '{{failed_step.id}}'"

    per_step_overrides:
      - step: "send_report"
        on_failure: "halt"    # Don't skip the final send — stop and alert me
```

## Wskazówki Dotyczące Budowania Przepływów Pracy

1. **Zacznij od wyzwalacza.** Co uruchamia przepływ pracy? Czas, zdarzenie, polecenie?
2. **Utrzymuj przepływy pracy skupione.** Jeden przepływ pracy, jedno zadanie. „Poranny Briefing" nie powinien też obsługiwać segregacji e-maili.
3. **Testuj za pomocą próbnych uruchomień.** Użyj `openclaw lobster run "My Workflow" --dry-run`, aby zobaczyć, co *by się stało*, bez faktycznego wykonania.
4. **Używaj etapu agenta swobodnie.** Za każdym razem, gdy potrzebujesz oceny, podsumowania lub przetwarzania języka naturalnego, użyj etapu agenta. To jest supermoc Lobster w porównaniu z tradycyjnymi narzędziami do automatyzacji.
5. **Konfiguruj powiadomienia o błędach.** Nie chcesz, aby przepływ pracy po cichu się nie powiódł przez dni. Zawsze konfiguruj alerty o awariach.

Lobster zamienia Twojego agenta z reaktywnego (odpowiadaj gdy zapytany) na proaktywny (rób rzeczy automatycznie). Najlepsze przepływy pracy to te, które konfigurujesz raz i o nich zapominasz — po cichu ułatwiając Ci życie w tle.
